# 09. Contest Submission Checklist

## 제출 파일 체크리스트

- [ ] 참가신청서
- [ ] 개인정보 수집 및 이용동의서
- [ ] 참가각서
- [ ] 발표자료
- [ ] 시각화 결과물

제출 형식:

- 참가신청서, 개인정보 수집 및 이용동의서는 서명 후 스캔하여 PDF로 제출한다.
- 발표자료는 PPTX 또는 PDF로 제출한다.
- 시각화 결과물은 GitHub Pages URL로 제출한다.

## 시각화 결과물 체크리스트

- [ ] GitHub Pages에서 웹 대시보드가 정상 접속된다.
- [ ] 서울 25개 구 지도가 표시된다.
- [ ] 구별 유모차 생활보행 점수가 표시된다.
- [ ] 250m 격자별 heatmap이 표시된다.
- [ ] 의료/행정/교육/여가 카테고리 필터가 동작한다.
- [ ] 구 클릭 시 상세 페이지로 이동한다.
- [ ] 구별 TOP 5 / BOTTOM 5가 표시된다.
- [ ] 카테고리별 구 순위가 표시된다.
- [ ] 벤치마킹 추천 페이지가 표시된다.
- [ ] 방법론 페이지에 산식과 데이터 출처가 정리되어 있다.
- [ ] 데이터 파일 누락 시 앱이 깨지지 않고 누락 파일명을 보여준다.
- [ ] `docs/data_schema_actual.md`와 `reports/data_validation_report.md`로 실제 산출물 스키마와 검증 결과를 확인했다.
- [ ] GitHub Pages URL에서 hash route 기반 내부 페이지가 정상 접근된다.
- [ ] 샘플 데이터, 더미 데이터, 임의 점수, 임의 시설 마커를 사용하지 않는다.

## 발표자료 체크리스트

- [ ] 프로젝트명 `잇다(:connect)`가 명확히 표시된다.
- [ ] 캐치프레이즈 `아이와 도시를 잇다. 서울의 미래를 잇다.`가 포함된다.
- [ ] 핵심 질문이 제시된다.
- [ ] 왜 250m 격자를 사용하는지 설명한다.
- [ ] 유모차 생활보행 점수 산식을 설명한다.
- [ ] 의료/행정/교육/여가 카테고리 구성을 설명한다.
- [ ] 서울 25개 구 비교 결과를 제시한다.
- [ ] 하위 구의 개선 여지와 벤치마킹 방향을 설명한다.
- [ ] 데이터 출처와 전처리 과정을 설명한다.
- [ ] 방법론 페이지의 실제 사용 데이터 목록이 `metadata.json`의 `source_datasets` 기준으로 표시된다.
- [ ] 한계와 주의사항을 명확히 설명한다.

## 서울 데이터 허브 서비스 활용 내용 체크리스트

- [ ] 서울시 자치구 경계 공간데이터 활용 여부를 기록한다.
- [ ] 서울시 행정동 경계 공간데이터 활용 여부를 기록한다.
- [ ] 병의원 또는 의료기관 위치 데이터 활용 여부를 기록한다.
- [ ] 주민센터, 구청, 시청 등 행정 시설 데이터 활용 여부를 기록한다.
- [ ] 어린이집, 유치원 위치 데이터 활용 여부를 기록한다.
- [ ] 공원, 도서관, 문화시설 데이터 활용 여부를 기록한다.
- [ ] Optional 데이터 사용 여부를 기록한다.
- [ ] 데이터별 출처 URL, 수집일, 라이선스 또는 이용 조건을 정리한다.
- [ ] 실제 사용하지 않은 원천 데이터 후보를 사용 데이터로 표시하지 않는다.
- [ ] 대형상업시설 데이터 미확보 시 제외 사실과 대체 산식을 설명한다.

## 심사 기준 대응 체크리스트

데이터 활용성:

- [ ] 서울시 공개데이터를 핵심 분석에 사용한다.
- [ ] 데이터 전처리 과정이 재현 가능하게 정리되어 있다.
- [ ] 원천 데이터와 전처리 산출물이 구분되어 있다.

분석 타당성:

- [ ] 점수 산식과 기준거리가 명확하다.
- [ ] 카테고리별 가중치가 문서화되어 있다.
- [ ] 구별 점수 집계 방식이 명확하다.
- [ ] Optional 데이터 적용 여부가 투명하게 기록되어 있다.

시각화 완성도:

- [ ] 지도, heatmap, 순위, 비교 차트가 서로 연결된다.
- [ ] 범례와 tooltip이 제공된다.
- [ ] 모바일과 데스크톱에서 핵심 정보가 겹치지 않는다.
- [ ] 데이터 없음 상태가 명확하다.

사회적 활용성:

- [ ] 아이동반 이동성 관점의 생활시설 접근성 격차를 설명한다.
- [ ] 낮은 점수 지역을 낙인찍지 않고 개선 여지로 표현한다.
- [ ] 벤치마킹 추천을 통해 실무적 참고 방향을 제시한다.

윤리와 한계:

- [ ] 실제 통행 가능 여부를 확정하지 않는다는 한계를 명시한다.
- [ ] 금지 표현을 사용하지 않는다.
- [ ] 공개데이터 기반 지표라는 점을 반복적으로 안내한다.
